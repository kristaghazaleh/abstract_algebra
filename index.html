Core Definitions

A relation on a set \(S\) is a subset of \(S\times S\). Writing \(a\sim b\) simply means that the pair \((a,b)\) lies in that subset. An equivalence relation is a relation that is reflexive, symmetric, and transitive. Once those three properties hold, the set is broken into equivalence classes, where \[ [a]=\{x\in S:x\sim a\}. \] These classes partition the set, meaning every element lies in exactly one class.

A binary operation on a set \(G\) is a map \(\ast:G\times G\to G\). The phrase “on \(G\)” already includes closure. A group is a set with a binary operation that is associative, has an identity element, and gives every element an inverse. If the operation also satisfies \(ab=ba\) for all \(a,b\in G\), then the group is abelian. A subgroup \(H\le G\) is a subset that is itself a group under the same operation. A group is cyclic if one element generates the whole group. In multiplicative notation, \[ \langle g\rangle=\{g^n:n\in\mathbb Z\}, \] and in additive notation, \[ \langle g\rangle=\{ng:n\in\mathbb Z\}. \]

In the language of maps, a homomorphism \(\varphi:G\to G'\) is a function satisfying \[ \varphi(xy)=\varphi(x)\varphi(y) \] for all \(x,y\in G\). Its kernel is \[ \ker\varphi=\{g\in G:\varphi(g)=e'\}, \] and its image is \[ \operatorname{im}\varphi=\{\varphi(g):g\in G\}. \] A subgroup \(N\le G\) is normal if \(gN=Ng\) for every \(g\in G\), or equivalently if \(gNg^{-1}=N\) for every \(g\in G\). Normality is exactly what allows one to form a quotient group \(G/N\).

Later in the course the language shifts from groups to rings. A ring \(R\) is a set with two operations, usually written \(+\) and \(\cdot\), such that \((R,+)\) is an abelian group, multiplication is associative, and multiplication distributes over addition. A ring is commutative if multiplication is commutative, and it has unity if there is an element \(1\) with \(1a=a1=a\) for all \(a\in R\). A unit is an element with a multiplicative inverse. A zero divisor is a nonzero element \(a\) for which there exists nonzero \(b\) with \(ab=0\). An integral domain is a commutative ring with unity and no zero divisors. A field is a commutative ring with unity in which every nonzero element is a unit.

Sets, Relations, Equivalence Classes, and Well-Definedness

How an equivalence relation works

When a problem defines a relation, the first thing to ask is whether it is actually an equivalence relation. That means checking three properties in order. Reflexive means every element is related to itself. Symmetric means if \(a\sim b\), then \(b\sim a\). Transitive means if \(a\sim b\) and \(b\sim c\), then \(a\sim c\). If even one of those fails, the relation is not an equivalence relation, and there is no point pretending that equivalence classes exist in the usual sense.

The standard model is congruence modulo \(n\) on \(\mathbb Z\). We write \[ a\equiv b\pmod n \] when \(n\mid(a-b)\). This relation is reflexive because \(n\mid(a-a)=0\), symmetric because divisibility of \(a-b\) is the same as divisibility of \(b-a\), and transitive because if \(n\mid(a-b)\) and \(n\mid(b-c)\), then \(n\mid(a-c)\). The resulting equivalence classes are exactly the residue classes modulo \(n\).

Another important example is the fraction construction for rational numbers. On pairs \((a,b)\) with \(b\neq 0\), define \[ (a,b)\sim(c,d)\quad\Longleftrightarrow\quad ad=bc. \] The point of this relation is that many different pairs represent the same rational number. For instance, \((1,2)\), \((2,4)\), and \((3,6)\) all belong to the same class. This example is where well defined becomes unavoidable.

What “well defined” is really saying

Suppose the objects in your new structure are equivalence classes, not ordinary elements. Then any operation or map you define on those classes must give the same answer no matter which representative you pick. That is exactly what “well defined” means. If it fails, then the rule is not a legitimate operation on the quotient object.

The basic test is always the same. Start by assuming two representatives describe the same class. Translate that into the defining relation. Then apply the proposed construction to both representatives and prove the outputs are equivalent. For example, on \(\mathbb Z_n\), multiplication is defined by \[ [a][b]=[ab]. \] To justify this, one must prove that if \(a\equiv a'\pmod n\) and \(b\equiv b'\pmod n\), then \(ab\equiv a'b'\pmod n\). This is not a side detail. It is the proof that multiplication on residue classes is legitimate.

Whenever a question mentions cosets, quotient groups, quotient rings, rational numbers built from pairs, or any operation on classes, the phrase “is this well defined?” should appear in your head immediately. In abstract algebra, many constructions look obvious until you remember that representatives are not unique.

Groups

What the group axioms are really doing

A group is a set where the operation behaves enough like addition or multiplication that algebraic manipulation makes sense. The order of the axioms matters. Before asking about associativity or inverses, make sure the operation is actually on the set. If the output can leave the set, then you do not even have a binary operation. After that, associativity lets you remove parentheses, the identity gives a neutral element, and inverses allow equations to be solved by cancellation.

The standard examples should feel automatic. The group \((\mathbb Z,+)\) has identity \(0\), and the inverse of \(n\) is \(-n\). The group \((\mathbb R^\times,\cdot)\) consists of the nonzero real numbers under multiplication; its identity is \(1\), and the inverse of \(x\) is \(1/x\). By contrast, \((\mathbb Z,\cdot)\) is not a group because most integers do not have multiplicative inverses in \(\mathbb Z\). This is an important habit: when a structure fails to be a group, name the exact axiom that fails.

Once you know a set is a group, several useful facts become automatic. The identity is unique. Each inverse is unique. Cancellation holds. The formula \[ (ab)^{-1}=b^{-1}a^{-1} \] works in every group, and \[ (a^{-1})^{-1}=a. \] These are not extra axioms; they are consequences you can prove from the axioms.

How to attack a “prove or disprove group” problem

If the problem asks you to prove something is a group, the clean route is usually: verify closure, mention associativity if it comes from a familiar ambient operation, identify the identity, and solve for the inverse of a general element. If the problem asks you to show something is not a group, go hunting for the earliest failure. Closure is often fastest. If closure survives, look for lack of identity or lack of inverses. There is no prize for checking all axioms once one of them already fails.

A good exam instinct is that binary-operation questions are often traps. A strange operation on a strange set may fail before anything interesting happens. So the first hidden question is always: does this operation really send the set back into itself?

Subgroups, Cyclic Groups, Generators, and Orders

Why the subgroup test matters

In principle, to prove \(H\le G\), you could recheck all the group axioms inside \(H\). In practice, that is almost never the right move. The standard subgroup test says that if \(H\neq\varnothing\) and for all \(a,b\in H\) one has \[ ab^{-1}\in H, \] then \(H\) is a subgroup of \(G\). This works because closure, the identity, and inverses all fall out of that one condition. In a finite setting there is an even shorter test: if \(H\) is nonempty, finite, and closed under the operation, then it is automatically a subgroup.

Cyclic groups are the easiest groups to understand because one element controls everything. If \(G=\langle g\rangle\), then every element looks like a power of \(g\) or, in additive notation, an integer multiple of \(g\). Every cyclic group is abelian, and every subgroup of a cyclic group is cyclic. This is why cyclic groups show up so often in examples: they are simple enough to compute with but rich enough to illustrate the main ideas.

How to think about the order of an element

The order of an element \(g\) is the least positive integer \(n\) such that \(g^n=e\), if such an integer exists. In additive notation, it is the least positive \(n\) such that \(ng=0\). The order of \(g\) is exactly the size of the subgroup \(\langle g\rangle\). So whenever you compute an element order, you are also computing the size of the cyclic subgroup it generates.

Some formulas are worth internalizing because they appear constantly. In \(\mathbb Z_n\), the order of \([k]\) is \[ \frac{n}{\gcd(n,k)}. \] In a finite cyclic group of order \(n\), the element \(a^k\) has order \[ \frac{n}{\gcd(n,k)}. \] For permutations, the order is the least common multiple of the lengths of the disjoint cycles. For direct products, the order is the least common multiple of the component orders.

Those formulas explain why generators of \(\mathbb Z_n\) are exactly the classes \([k]\) with \(\gcd(k,n)=1\). They also explain why a group of prime order is automatically cyclic: every nonidentity element must have order dividing the prime, so its order must be that prime.

Permutation Groups, Cycle Notation, Alternating Groups, and Dihedral Groups

Why cycle notation is the right language

A permutation is a bijection from a set to itself, and the group \(S_n\) consists of all permutations of \(\{1,2,\dots,n\}\). Two habits matter immediately. First, composition is read right to left. Second, cycle notation is almost always better than two-line notation because it reveals structure. A cycle such as \((1\,3\,5)\) means that \(1\) goes to \(3\), \(3\) goes to \(5\), \(5\) goes to \(1\), and everything else is fixed.

Disjoint cycles commute, and that fact is what makes permutation order easy to compute. If \[ \sigma=(1\,4\,2)(3\,5), \] then the order of \(\sigma\) is \[ \operatorname{lcm}(3,2)=6. \] The reason is that one part of the permutation resets after three steps and the other resets after two, so the whole permutation resets when both have returned to where they started.

Transpositions, parity, and \(A_n\)

A transposition is a 2-cycle. Every permutation can be written as a product of transpositions. Even though the factorization is not unique, the parity of the number of transpositions is unique. That is why every permutation is either even or odd. The set of even permutations forms the alternating group \(A_n\), which has index \(2\) in \(S_n\), so \[ |A_n|=\frac{n!}{2}. \]

This is also a useful source of normal subgroups: \(A_n\) is the kernel of the sign homomorphism \[ \operatorname{sgn}:S_n\to\{\pm 1\}, \] so \(A_n\trianglelefteq S_n\).

Dihedral groups as symmetry groups

The dihedral group \(D_n\) is the symmetry group of a regular \(n\)-gon. It contains \(n\) rotations and \(n\) reflections, so \[ |D_n|=2n. \] A standard presentation is \[ D_n=\langle r,s\mid r^n=e,\ s^2=e,\ srs=r^{-1}\rangle. \] Every element can be written in the form \(r^k\) or \(sr^k\). The rotations form a cyclic subgroup \(\langle r\rangle\) of order \(n\), and each reflection has order \(2\).

Conceptually, \(D_n\) is important because it is the bridge between algebra and geometry. It gives you a concrete nonabelian group where you can still visualize what the elements do.

Cosets, Equivalence by a Subgroup, and Lagrange's Theorem

Why cosets matter

If \(H\le G\), a left coset is a set of the form \[ gH=\{gh:h\in H\}, \] and a right coset is a set of the form \[ Hg=\{hg:h\in H\}. \] At first these look like arbitrary translated copies of \(H\), but they do something fundamental: they partition the whole group. Two left cosets are either identical or disjoint, and every element of \(G\) belongs to exactly one left coset of \(H\).

The reason cosets are so powerful is that each one has the same number of elements as \(H\). Multiplication by \(g\) gives a bijection \[ H\to gH. \] So cosets are a counting tool. Once a subgroup exists, the group is chopped into equally sized pieces.

Lagrange's theorem and how to use it

If \(G\) is finite and \(H\le G\), then \[ |G|=[G:H]\cdot |H|. \] This is Lagrange’s theorem. The immediate consequence is that \(|H|\) divides \(|G|\). Since the order of an element is the size of the cyclic subgroup it generates, the order of every element also divides \(|G|\).

This theorem is used more often to rule things out than to build things. If a number does not divide \(|G|\), then there cannot be a subgroup of that order and there cannot be an element of that order. But Lagrange’s theorem is only a necessary condition, not a sufficient one. A divisor of \(|G|\) does not automatically occur as an element order or subgroup order in every group.

One especially useful corollary is that a subgroup of index \(2\) is always normal. There are only two left cosets and two right cosets, so they must agree. This fact shows up constantly in quotient problems.

Direct Products and Finite Abelian Group Structure

How direct products behave

If \(G\) and \(H\) are groups, then the direct product \(G\times H\) is the set of ordered pairs with componentwise operation: \[ (g_1,h_1)(g_2,h_2)=(g_1g_2,h_1h_2). \] The identity is \((e_G,e_H)\), and inverses are taken componentwise. This construction is simple, but it is important because many groups decompose into products of easier pieces.

The order of \((g,h)\) is the least common multiple of the orders of \(g\) and \(h\), provided those orders are finite. This is the right way to test whether a direct product might be cyclic. For example, \[ \mathbb Z_m\times\mathbb Z_n \] is cyclic if and only if \(\gcd(m,n)=1\). If the factors share a common prime factor, then no element can have the full order \(mn\).

Why this matters for finite abelian groups

Finite abelian groups are built from cyclic prime-power pieces. In practice, that means factor the group order into prime powers and then break those prime powers into cyclic factors. For example, groups of order \(8\) can look like \[ \mathbb Z_8,\qquad \mathbb Z_4\times\mathbb Z_2,\qquad \mathbb Z_2\times\mathbb Z_2\times\mathbb Z_2. \] These groups have the same size, but they are not isomorphic because their element orders are different.

This viewpoint is also a reliable way to disprove isomorphisms. Two finite groups cannot be isomorphic if one is cyclic and the other is not, if one is abelian and the other is not, or if they have different numbers of elements of a given order.

Group Homomorphisms, Kernels, Images, and Isomorphisms

The homomorphism law is the whole definition

A homomorphism is a map that respects the group operation. That is all it means. If \[ \varphi:G\to G' \] satisfies \[ \varphi(xy)=\varphi(x)\varphi(y) \] for all \(x,y\in G\), then \(\varphi\) is a homomorphism. Once this is true, several useful consequences become automatic: \[ \varphi(e_G)=e_{G'},\qquad \varphi(x^{-1})=\varphi(x)^{-1},\qquad \varphi(x^n)=\varphi(x)^n. \] The point is that a homomorphism preserves algebraic structure, not necessarily the everyday arithmetic operations you are used to from \(\mathbb R\).

The two objects that control a homomorphism are its kernel and image. The kernel tells you what collapses to the identity. The image tells you what part of the codomain is actually reached. The image is always a subgroup of the codomain, and the kernel is always a normal subgroup of the domain. That last statement is one of the biggest reasons kernels matter: normal subgroups are exactly the subgroups that arise naturally from maps.

Injective, surjective, and isomorphic

The cleanest criterion for injectivity is \[ \varphi \text{ is injective } \Longleftrightarrow \ker\varphi=\{e\}. \] This is one of the most useful short theorems in the course. It means that whenever you want to prove a homomorphism is one-to-one, you usually try to compute the kernel first.

A homomorphism is surjective exactly when its image is the whole codomain. If a homomorphism is both injective and surjective, then it is an isomorphism. Isomorphic groups are algebraically the same. They preserve element orders, cyclicity, abelian behavior, subgroup structure, and many other invariants. So to prove two groups are not isomorphic, look for a structural mismatch rather than trying to argue directly with every element.

The most efficient way to prove an isomorphism is to define a candidate map, verify the homomorphism law, and then prove bijectivity. The most efficient way to disprove one is to compare invariants.

Normal Subgroups and Quotient Groups

Why normality is the right condition

A subgroup \(N\le G\) is normal if \[ gN=Ng \] for every \(g\in G\), or equivalently if \[ gNg^{-1}=N \] for every \(g\in G\). In an abelian group every subgroup is normal, but in a nonabelian group this is a real restriction. The reason normality matters is that it is exactly what makes multiplication of cosets work.

If \(N\trianglelefteq G\), then the set of cosets \(G/N\) becomes a group under the rule \[ (aN)(bN)=(ab)N. \] Without normality, different representatives can give different answers, so the product would not be well defined. This is one of the central conceptual points in the course: quotient groups are not formed from arbitrary subgroups. They are formed from normal subgroups because normality is the compatibility condition that makes the quotient behave like a group.

How to think about the quotient

The quotient \(G/N\) should be thought of as collapsing everything in the same coset into a single object. The subgroup \(N\) itself becomes the identity coset. The inverse of \(aN\) is \(a^{-1}N\), and if \(G\) is finite then \[ |G/N|=[G:N]=\frac{|G|}{|N|}. \] In a quotient, elements that differ by something in \(N\) are treated as indistinguishable.

This is why quotients and homomorphisms are so closely tied. A homomorphism identifies elements that differ by something in its kernel, and the quotient by that kernel packages exactly that identification into a new group.

Group Actions

A group action is the formal way to say that a group moves points of a set. This is where abstract algebra becomes concrete. A symmetry of a polygon, a permutation of symbols, a matrix acting on vectors, and a group acting on its own cosets are all examples of the same idea.

The definition and what it means

An action of \(G\) on a set \(X\) is a map \[ \cdot:G\times X\to X \] such that \[ e\cdot x=x \qquad\text{and}\qquad (g_1g_2)\cdot x=g_1\cdot(g_2\cdot x) \] for all \(x\in X\) and all \(g_1,g_2\in G\). The first axiom says the identity does nothing. The second says that acting by \(g_2\) and then by \(g_1\) is the same as acting by the product \(g_1g_2\).

Every action gives a homomorphism \[ \Phi:G\to S_X \] into a symmetric group, because each \(g\in G\) determines a permutation of \(X\) by \(x\mapsto g\cdot x\). So group actions are really permutation representations in disguise.

Orbits and stabilizers

For a point \(x\in X\), the orbit is \[ Gx=\{g\cdot x:g\in G\}, \] and the stabilizer is \[ G_x=\{g\in G:g\cdot x=x\}. \] The stabilizer is always a subgroup. The orbit is the set of all places the point can be moved by the action. These two objects are linked by the orbit-stabilizer theorem: \[ |Gx|=[G:G_x]=\frac{|G|}{|G_x|} \] whenever \(G\) is finite.

Conceptually, this theorem converts motion into counting. If you can understand what fixes a point, you can count how many places that point can move.

Quotient Continuation

The natural projection

If \(N\trianglelefteq G\), then the map \[ \pi:G\to G/N,\qquad \pi(a)=aN \] is a homomorphism. It is called the natural projection. Its kernel is exactly \(N\), because \[ \pi(a)=N \Longleftrightarrow aN=N \Longleftrightarrow a\in N. \] This already shows that every normal subgroup arises as a kernel.

The first isomorphism theorem

If \(\varphi:G\to G'\) is a homomorphism with kernel \(H\), then \[ G/H\cong \varphi(G). \] This theorem explains what quotients are for. The quotient by the kernel removes exactly the information that the homomorphism forgets, and what remains is isomorphic to the image.

The usual map is \[ \mu:G/H\to \varphi(G),\qquad \mu(aH)=\varphi(a). \] The real work is showing this is well defined. Once that is done, it becomes a routine homomorphism and bijection proof.

A classic example is the map \[ \mathbb Z\to \mathbb Z_n,\qquad k\mapsto [k]. \] Its kernel is \(n\mathbb Z\), so \[ \mathbb Z/n\mathbb Z\cong \mathbb Z_n. \] This is the abstract explanation for ordinary modular arithmetic.

Rings

What changes when you move from groups to rings

In group theory there is one operation. In ring theory there are two. Addition behaves like an abelian group, while multiplication behaves like an associative operation that distributes over addition. The main point of a ring is that addition and multiplication interact in a controlled way.

A ring \(R\) satisfies three big requirements. First, \((R,+)\) is an abelian group. Second, multiplication is associative. Third, multiplication distributes over addition: \[ a(b+c)=ab+ac,\qquad (a+b)c=ac+bc. \] If multiplication is commutative, the ring is commutative. If there is an element \(1\) such that \(1a=a1=a\) for all \(a\), then the ring has unity.

Standard examples include \(\mathbb Z\), \(\mathbb Q\), \(\mathbb R\), and \(\mathbb C\). Matrix rings are also rings, but usually not commutative. The residue-class rings \(\mathbb Z_n\) are among the most important examples in this course because they show how units and zero divisors interact in a concrete setting.

How to prove something is a ring

When you are asked to show a set is a ring, it is usually wasteful to redo every axiom from scratch if the operations come from a familiar ambient ring. The right strategy is to use the ambient structure. Show the set is closed under addition and multiplication, show additive inverses stay in the set, and then state that associativity and distributivity are inherited from the larger ring. This is exactly the same proof pattern used in subgroup questions.

Units, Zero Divisors, Integral Domains, and Fields

How these notions fit together

In a ring with unity, a unit is an element with a multiplicative inverse. A zero divisor is a nonzero element \(a\) for which there exists nonzero \(b\) with \(ab=0\). These two ideas pull in opposite directions. Units behave like elements you can safely divide by. Zero divisors are exactly the elements that make cancellation fail.

An integral domain is a commutative ring with unity and no zero divisors. This is the setting where cancellation works for multiplication. A field is even stronger: every nonzero element is a unit. So every field is an integral domain. In a finite commutative ring with unity, the converse also holds: every finite integral domain is a field.

The group of units of a ring is important enough to deserve its own notation: \[ R^\times=\{a\in R:a\text{ is a unit}\}. \] This set forms a group under multiplication. The proof mirrors familiar group proofs: closure comes from multiplying inverses in the reverse order, the identity is \(1\), and the inverse of a unit is again a unit.

The special case of \(\mathbb Z_n\)

In \(\mathbb Z_n\), the key invariant is \(\gcd(a,n)\). A class \([a]\) is a unit if and only if \[ \gcd(a,n)=1. \] If \(\gcd(a,n)\neq 1\), then \([a]\) is not a unit, and in fact a nonzero such class is a zero divisor. So in \(\mathbb Z_n\), being a unit, being relatively prime to \(n\), and avoiding zero-divisor behavior are all part of the same picture.

This explains why \(\mathbb Z_p\) is a field when \(p\) is prime. Every nonzero class modulo \(p\) is relatively prime to \(p\), so every nonzero class is a unit.

Ring Homework Examples

How to write cleaner ring proofs

Many ring homework problems look longer than they really are. They often ask you to verify one axiom of a construction that is already almost finished. In that situation, the cleanest proof is to write down the definition of the operation, compute both sides, and then identify the ring axiom in the underlying structure that makes them equal.

For example, in the field of quotients of an integral domain, addition is defined by \[ [(a,b)]+[(c,d)]=[(ad+bc,bd)]. \] To prove commutativity, compute \[ [(a,b)]+[(c,d)]=[(ad+bc,bd)] \] and \[ [(c,d)]+[(a,b)]=[(cb+da,db)]. \] Then point out that \(ad+bc=cb+da\) and \(bd=db\) in the domain. Nothing deeper is needed.

The same writing style works for distributivity, multiplicative identity, and multiplicative inverses. In each case, start from the definition, simplify using the ring laws in the background, and then translate the result back into the quotient notation.

For polynomial-ring proofs, the pattern is similar. If you want to prove multiplication is associative in \(R[x]\), compare coefficients of \(x^n\) on both sides. If you want to prove \(R[x]\) is an integral domain when \(R\) is, look at the leading coefficients of two nonzero polynomials. The product cannot vanish because the product of the leading coefficients is nonzero in an integral domain.

Modular Arithmetic

Units and zero divisors in \(\mathbb Z_n\)

The ring \(\mathbb Z_n\) is where ring theory becomes concrete enough to compute by hand. For a nonzero class \([a]\in \mathbb Z_n\), the central fact is \[ [a]\in \mathbb Z_n^\times \qquad\Longleftrightarrow\qquad \gcd(a,n)=1. \] This comes from Bézout’s identity. If \(\gcd(a,n)=1\), then there exist integers \(u,v\) such that \[ au+nv=1, \] so modulo \(n\), \[ au\equiv 1\pmod n. \] Thus \([u]\) is the inverse of \([a]\).

If \(\gcd(a,n)=d>1\), then \([a]\) is not a unit. In fact, it becomes a zero divisor. The point is that \[ a\cdot \frac{n}{d} \] is divisible by \(n\), so \([a]\left[\frac{n}{d}\right]=[0]\), even though both classes are nonzero mod \(n\). This is why in \(\mathbb Z_n\), nonzero zero divisors are exactly the nonunits.

Solving \(ax\equiv b\pmod n\)

The equation \[ ax\equiv b\pmod n \] splits into two cases. If \(\gcd(a,n)=1\), then \([a]\) is a unit and there is a unique solution: \[ x\equiv a^{-1}b\pmod n. \] If \(\gcd(a,n)=d>1\), then solutions exist if and only if \(d\mid b\). When they do exist, there are exactly \(d\) distinct solutions modulo \(n\).

The practical method is to divide the entire congruence by \(d\), solve the reduced congruence modulo \(n/d\), and then lift the solution back to the original modulus. This is the systematic version of what you are often doing intuitively in examples.

Fermat and Euler

If \(p\) is prime and \(a\not\equiv 0\pmod p\), then Fermat’s little theorem says \[ a^{p-1}\equiv 1\pmod p. \] Euler’s theorem generalizes this. If \(\gcd(a,n)=1\), then \[ a^{\varphi(n)}\equiv 1\pmod n, \] where \(\varphi(n)\) is the number of units in \(\mathbb Z_n\). These theorems are really statements about powers inside the finite group of units \(\mathbb Z_n^\times\).

Fields of Quotients

Why the construction is needed

An integral domain lets you multiply without zero divisors, but it may still lack division. The field of quotients is the standard way to fix that. Starting from an integral domain \(D\), one builds a field \(F\) that contains \(D\) and in which every nonzero element of \(D\) becomes invertible. The rational numbers \(\mathbb Q\) built from \(\mathbb Z\) are the model example.

The construction begins with pairs \((a,b)\in D\times D\) with \(b\neq 0\). Think of \((a,b)\) as the formal fraction \(a/b\). Two such pairs should represent the same fraction when \[ ad=bc. \] So define \[ (a,b)\sim(c,d)\quad\Longleftrightarrow\quad ad=bc. \] The field of quotients consists of the equivalence classes \([(a,b)]\).

The arithmetic and the embedding of \(D\)

Addition and multiplication are defined by the familiar fraction rules: \[ [(a,b)]+[(c,d)]=[(ad+bc,bd)], \qquad [(a,b)]\cdot[(c,d)]=[(ac,bd)]. \] The first real job in the construction is to prove that these are well defined. Once that is done, the ring axioms follow by computation in the domain \(D\). The additive identity is \([(0,1)]\), the multiplicative identity is \([(1,1)]\), and every nonzero element \([(a,b)]\) has inverse \([(b,a)]\) because \(a\neq 0\) in that case.

The domain \(D\) sits inside its field of quotients through the map \[ i:D\to F,\qquad i(a)=[(a,1)]. \] This is an injective ring homomorphism, so one usually stops distinguishing \(a\) from \([(a,1)]\). That is why \(\mathbb Z\) is viewed naturally as a subring of \(\mathbb Q\).

Examples worth knowing

The field of quotients of \(\mathbb Z\) is \(\mathbb Q\). The field of quotients of \[ D=\{n+m\sqrt 2:n,m\in\mathbb Z\} \] is \[ \mathbb Q(\sqrt 2)=\{a+b\sqrt 2:a,b\in\mathbb Q\}. \] The clean way to prove this is to rationalize \[ \frac{n+m\sqrt 2}{r+s\sqrt 2} \] by multiplying top and bottom by \(r-s\sqrt 2\).

Conceptually, the field of quotients is unique up to isomorphism. So even if someone constructs it a different way, it is still “the same” field from the algebraic point of view.

Polynomial Rings

Polynomials are formal objects

Given a ring \(R\), a polynomial in \(R[x]\) is a formal finite sum \[ a_0+a_1x+\cdots+a_nx^n \] with coefficients in \(R\). The word formal matters. The symbol \(x\) is not yet a number. It is an indeterminate. This is why two different-looking polynomials can define the same function on a finite ring without being the same polynomial. For example, in \(\mathbb Z_2[x]\), \[ x^2+x \] vanishes on both elements of \(\mathbb Z_2\), but it is not the zero polynomial.

The degree of a nonzero polynomial is the largest exponent with nonzero coefficient. The zero polynomial is usually assigned degree \(-1\) or left undefined, depending on the convention. Addition in \(R[x]\) is coefficientwise, and multiplication is defined by collecting like powers: \[ \left(\sum a_ix^i\right)\left(\sum b_jx^j\right) =\sum_k\left(\sum_{i+j=k}a_ib_j\right)x^k. \] With these operations, \(R[x]\) is a ring. If \(R\) is commutative, then \(R[x]\) is commutative. If \(R\) has unity, then so does \(R[x]\).

Evaluation and roots

If \(R\) is commutative and \(\alpha\in R\), the evaluation map \[ \varphi_\alpha:R[x]\to R,\qquad \varphi_\alpha(f)=f(\alpha), \] is a ring homomorphism. More generally, if \(R\subseteq R'\) and \(\alpha\in R'\), evaluation still makes sense as a map into \(R'\). An element \(\alpha\) is a root of \(f(x)\) if \[ f(\alpha)=0. \] This is how polynomials connect to field extensions: sometimes a polynomial has no root in the original field and one must enlarge the field to find one.

Division algorithm and its consequences

If \(F\) is a field and \(g(x)\neq 0\) in \(F[x]\), then for every \(f(x)\in F[x]\) there exist unique polynomials \(q(x)\) and \(r(x)\) such that \[ f(x)=q(x)g(x)+r(x), \qquad \deg r<\deg g. \] This is the polynomial division algorithm. The reason the coefficient ring must be a field is that one needs to divide by the leading coefficient of \(g\).

A direct consequence is the factor theorem: if \(a\in F\) and \(f(a)=0\), then \[ x-a \] divides \(f(x)\). Repeating this argument shows that a nonzero polynomial of degree \(n\) over a field has at most \(n\) roots in that field.

This fact has surprisingly strong consequences. For instance, if \(F\) is a field and \(G\) is a finite subgroup of \(F^\times\), then every element of \(G\) is a root of \[ x^{|G|}-1. \] Since that polynomial has degree \(|G|\), it cannot have more than \(|G|\) roots. This is one ingredient in the proof that finite subgroups of the multiplicative group of a field are cyclic.

Proof Patterns

A relation proof should start by naming the three properties you are checking. A subgroup proof should almost always use the subgroup test rather than rebuilding the group axioms from scratch. A homomorphism proof should begin with \(\varphi(xy)\) and compare it to \(\varphi(x)\varphi(y)\). A quotient proof should begin by checking normality or well-definedness before any computation with cosets. In ring theory, many proofs reduce to saying that closure is internal while associativity or distributivity is inherited from the ambient ring.

The main idea is that abstract algebra rewards the right first sentence. If the first sentence identifies the correct structure and theorem, the rest of the proof often becomes short and clean. If the first sentence is vague, the proof tends to become long and unfocused.

Counterexamples

A good counterexample is not just an example where the conclusion fails. It is an example that satisfies the hypotheses and then breaks the conclusion in the cleanest possible way. For a false universal statement, one example is enough. For a failed definition, one broken axiom is enough.

In this course the best counterexamples are usually small ones. To disprove an equivalence relation, break one of reflexive, symmetric, or transitive with explicit elements. To disprove that something is a group, find one failure of closure, identity, inverse, or associativity. To disprove normality, compute one conjugate that leaves the subgroup. To disprove an isomorphism, compare an invariant such as element orders or abelian behavior. The point is not to say everything that is wrong. The point is to show enough to force the statement to collapse.

Common Mistakes

The most common mistake in quotient problems is forgetting that operations on classes must be well defined. The most common mistake in subgroup problems is checking closure and stopping too early. The most common mistake in permutation problems is composing left to right instead of right to left. The most common mistake in homomorphism problems is confusing the kernel with the image, or assuming a homomorphism must be injective. The most common mistake in Lagrange-type arguments is treating divisibility as an existence theorem. It is not.

A useful correction habit is to ask, before writing anything long: what is the exact definition here, and what is the exact theorem that matches it? That one pause prevents a surprising number of algebra mistakes.
