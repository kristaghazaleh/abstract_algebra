Overview

This page is meant to work as plain study material and a long-term reference. The recurring theme in abstract algebra is that most problems are not solved by brute force computation. They are solved by knowing exactly which definition applies, which standard theorem is relevant, and what must be proved in what order. A lot of the work is pattern recognition: equivalence relation questions ask for three properties, subgroup questions ask for a closure-plus-inverse test, homomorphism questions ask for the operation-preserving law, and quotient questions ask whether normality and well-definedness have been handled correctly.

What to think first

  • Relation problem: check reflexive, symmetric, transitive, then identify the classes.
  • Operation problem: ask whether the operation is actually binary, then closure, associativity, identity, inverses.
  • Subset problem: ask whether this is a subgroup and use a subgroup test instead of redoing all axioms.
  • Order problem: think powers, divisibility, Lagrange, and least positive exponent.
  • Permutation problem: rewrite in disjoint cycles and compose right to left.
  • Map problem: compute \(\varphi(xy)\) and \(\varphi(x)\varphi(y)\) directly.
  • Quotient problem: ask whether the subgroup is normal and whether the operation on cosets is well defined.

Very short checklist of facts that solve a lot of exercises

  • Kernels are normal subgroups.
  • Images of homomorphisms are subgroups.
  • Every subgroup of an abelian group is normal.
  • A subgroup of index \(2\) is normal.
  • Every cyclic group is abelian, and every subgroup of a cyclic group is cyclic.
  • The order of an element divides the order of a finite group.
  • A group of prime order is cyclic.
  • Two groups cannot be isomorphic if they differ on abelian/nonabelian behavior, element orders, subgroup counts, or cyclicity.

Sets, Relations, Equivalence Classes, and Well-Definedness

Equivalence relations

A relation \(\sim\) on a set \(S\) is an equivalence relation if it is reflexive, symmetric, and transitive. These three properties are not optional details; they are exactly what makes it possible to group elements into classes that behave like the same object for the purpose of the problem.

  • Reflexive: for every \(a\in S\), show \(a\sim a\).
  • Symmetric: if \(a\sim b\), show \(b\sim a\).
  • Transitive: if \(a\sim b\) and \(b\sim c\), show \(a\sim c\).

Equivalence classes

The equivalence class of \(a\) is \([a]=\{x\in S : x\sim a\}\). Distinct equivalence classes are disjoint, and the equivalence classes partition the set. That means every element belongs to exactly one class. In practice, many algebraic constructions are built from equivalence classes rather than raw elements.

Standard examples

  • Congruence modulo \(n\): \(a\equiv b \pmod n\) iff \(n\mid(a-b)\).
  • Same absolute value on \(\mathbb{R}\): \(x\sim y\) iff \(|x|=|y|\).
  • Rational-number construction from integer pairs: \((a,b)\sim(c,d)\) iff \(ad=bc\).

Common examples to know cold

  • On \(\mathbb{Z}\), the relation \(a\sim b\iff a\equiv b\pmod 5\) has exactly five classes: \([0],[1],[2],[3],[4]\).
  • On \(\mathbb{R}\), the relation \(x\sim y\iff x-y\in\mathbb{Z}\) packages real numbers by their fractional part. A typical class is \([0.3]=\{\dots,-1.7,-0.7,0.3,1.3,2.3,\dots\}\).
  • For fractions, \((1,2)\sim(2,4)\sim(3,6)\) because they represent the same rational number. This is the standard model for why well-definedness matters.

Well-defined operations

When the elements of the new structure are equivalence classes, you must prove that any operation on those classes does not depend on the representative chosen. This is what well defined means. It is one of the most important proof ideas in the course. For example, on \(\mathbb{Z}_n\), to define multiplication by \([a][b]=[ab]\), you must show that if \(a\equiv a'\pmod n\) and \(b\equiv b'\pmod n\), then \(ab\equiv a'b'\pmod n\). If that fails, then the operation is not legitimate on classes.

How to prove well defined

  • Start by assuming two representatives describe the same class.
  • Translate that assumption into the defining relation.
  • Apply the proposed operation to both representatives.
  • Show the outputs are equivalent under the same relation.

Groups

Definition

A group is a set \(G\) together with a binary operation \(*\) such that closure, associativity, identity, and inverses all hold. If the operation also satisfies \(ab=ba\) for all \(a,b\in G\), the group is abelian.

Binary operation and closure

The first hidden issue in many problems is whether the proposed operation is even binary. The output must stay in the set. A set with an operation that leaves the set is not a group before any deeper axiom is even considered.

Basic facts used constantly

  • The identity element is unique.
  • Each element has a unique inverse.
  • Left and right cancellation hold.
  • \((ab)^{-1}=b^{-1}a^{-1}\).
  • \((a^{-1})^{-1}=a\).
  • If \(a^2=e\) for every \(a\in G\), then \(G\) is abelian.
  • If \(x^2=x\), then \(x=e\).

How to show something is not a group

  • Find one failure of closure.
  • Give one concrete failure of associativity.
  • Show no identity exists.
  • Show some element has no inverse in the set.

Common examples

  • \((\mathbb{Z},+)\), \((\mathbb{R},+)\), and \((\mathbb{C}^\times,\cdot)\) are groups.
  • \((\mathbb{Z},\cdot)\) is not a group because most elements have no multiplicative inverse in \(\mathbb{Z}\).
  • Invertible matrices form a group under multiplication, but all matrices do not.

Common examples that show up on tests

  • \((2\mathbb{Z},+)\) is a group under addition even though it is not all of \(\mathbb{Z}\). The identity is still \(0\), and the inverse of \(2k\) is \(-2k\).
  • \((\mathbb{Z}_n,+)\) is always a finite cyclic group generated by \([1]\).
  • \((U(n),\cdot)\), the units mod \(n\), is a group under multiplication mod \(n\). This is a common place where closure and inverses have to be checked in modular arithmetic.
  • \((\mathbb{R}\setminus\{0\},\cdot)\) is a group, but \((\mathbb{R},\cdot)\) is not because \(0\) has no inverse.

Subgroups, Cyclic Groups, Generators, and Orders

Subgroup tests

A subgroup is a subset that is itself a group under the same operation. In proofs, the most efficient test is usually: show the subset is nonempty and closed under \(ab^{-1}\). In additive notation, that becomes closure under subtraction.

  • Two-step subgroup test: \(H\neq\varnothing\) and for all \(a,b\in H\), \(ab^{-1}\in H\).
  • Finite subgroup test: if \(H\) is a nonempty finite subset closed under the operation, then \(H\le G\).
  • Additive version: show \(0\in H\) and \(a-b\in H\) whenever \(a,b\in H\).

Cyclic groups and generators

A group is cyclic if one element generates the whole group. In multiplicative notation, \(\langle a\rangle = \{a^n:n\in\mathbb{Z}\}\). In additive notation, \(\langle a\rangle = \{na:n\in\mathbb{Z}\}\). Cyclic groups are among the easiest groups to analyze because their structure is completely controlled by one generator.

  • Every cyclic group is abelian.
  • Every subgroup of a cyclic group is cyclic.
  • If \(|a|=n\), then \(|a^k|=n/\gcd(n,k)\).
  • In \(\mathbb{Z}_n\), the generator \([k]\) exists exactly when \(\gcd(k,n)=1\).
  • A finite cyclic group has exactly one subgroup of order \(d\) for each divisor \(d\) of \(n\).

Order of an element

The order of an element \(a\) is the least positive integer \(n\) such that \(a^n=e\), if such an \(n\) exists. If no such positive integer exists, then \(a\) has infinite order. In additive notation, the order of \(a\) is the least positive \(n\) with \(na=0\).

Ways to compute orders

  • In a cyclic group of order \(n\), use \(|a^k|=n/\gcd(n,k)\).
  • In \(\mathbb{Z}_n\), the order of \([k]\) is \(n/\gcd(n,k)\).
  • For permutations, use disjoint cycles and take the least common multiple of cycle lengths.
  • For product groups, take the least common multiple of component orders.

Useful consequences

  • If a finite group has no proper nontrivial subgroups, then it is cyclic of prime order.
  • If \(|G|\) is prime, then every nonidentity element generates \(G\).
  • A subgroup generated by an element always has size equal to the order of that element.

Common exam examples

  • In \(\mathbb{Z}_{12}\), the element \([8]\) has order \(12/\gcd(12,8)=3\), so \(\langle [8]\rangle=\{[0],[8],[4]\}\).
  • In \(\mathbb{Z}_{18}\), the generators are exactly the classes \([k]\) with \(\gcd(k,18)=1\), namely \([1],[5],[7],[11],[13],[17]\).
  • In a group of order \(15\), any element has order dividing \(15\), so possible element orders are \(1,3,5,15\).

Permutation Groups, Cycle Notation, Alternating Groups, and Dihedral Groups

Permutation basics

A permutation of a set \(A\) is a bijection from \(A\) to itself. The permutations of \(\{1,2,\dots,n\}\) form the symmetric group \(S_n\) under composition. Composition is read right to left: apply the rightmost permutation first.

  • Rewrite into cycle notation whenever possible.
  • Disjoint cycles commute.
  • The inverse of \((a_1\,a_2\,\dots\,a_k)\) is \((a_k\,\dots\,a_2\,a_1)\).
  • The order of a permutation is the least common multiple of the lengths of its disjoint cycles.

Transpositions and parity

A transposition swaps two entries and fixes the rest. Every permutation can be written as a product of transpositions. The parity of the number of transpositions used is invariant, so every permutation is either even or odd. The set of even permutations is the alternating group \(A_n\), a subgroup of \(S_n\) of index \(2\).

  • A \(k\)-cycle can be written as \(k-1\) transpositions.
  • A cycle of odd length is an even permutation iff \(k-1\) is even.
  • \(A_n\) is normal in \(S_n\) because it is the kernel of the sign homomorphism.

Conjugation in \(S_n\)

If \(\mu=(x_1\,x_2\,\dots\,x_k)\) and \(\sigma\in S_n\), then \[ \sigma\mu\sigma^{-1}=(\sigma(x_1)\,\sigma(x_2)\,\dots\,\sigma(x_k)). \] Conjugation preserves cycle structure. That means two permutations are conjugate in \(S_n\) exactly when they have the same cycle type.

Dihedral groups

The dihedral group \(D_n\) is the symmetry group of a regular \(n\)-gon. It has \(n\) rotations and \(n\) reflections, so \(|D_n|=2n\). A standard presentation is \[ D_n=\langle r,s\mid r^n=e,\ s^2=e,\ srs=r^{-1}\rangle. \] Every element can be written either as \(r^k\) or \(sr^k\).

Structure facts used a lot

  • \(S_n\) is nonabelian for \(n\ge 3\).
  • \(A_n\) has order \(n!/2\).
  • \(\langle r\rangle\) is a cyclic subgroup of \(D_n\) of order \(n\).
  • Reflections in \(D_n\) have order \(2\).

Common exam examples

  • \((1\,2\,3)(1\,2)=(1\,3)\) when composition is read right to left.
  • The order of \((1\,4\,2)(3\,5)\) is \(\operatorname{lcm}(3,2)=6\).
  • The permutation \((1\,2\,3)\) is even because \((1\,2\,3)=(1\,3)(1\,2)\), a product of two transpositions.
  • In \(D_4\), the rotation subgroup \(\langle r\rangle=\{e,r,r^2,r^3\}\) has index \(2\), so it is normal.
  • In \(D_n\), every element has the form \(r^k\) or \(sr^k\), and \((sr^k)^2=e\), so every reflection has order \(2\).

Cosets, Equivalence by a Subgroup, and Lagrange's Theorem

Cosets

If \(H\le G\) and \(a\in G\), the left coset is \(aH=\{ah:h\in H\}\) and the right coset is \(Ha=\{ha:h\in H\}\). Cosets are the subgroup version of equivalence classes. In fact, the relation \(a\sim b\iff a^{-1}b\in H\) is an equivalence relation, and its equivalence classes are exactly the left cosets of \(H\).

  • \(aH=bH\iff a^{-1}b\in H\).
  • Any two left cosets are either equal or disjoint.
  • Every left coset has the same size as \(H\).
  • The number of left cosets is the index \([G:H]\).

Lagrange's theorem

If \(G\) is finite and \(H\le G\), then \[ |G|=[G:H]|H|. \] So the order of every subgroup divides the order of the whole group.

Consequences that get used constantly

  • The order of any element divides \(|G|\).
  • If \(|G|\) is prime, then \(G\) is cyclic.
  • If \(H\) has index \(2\), then \(H\trianglelefteq G\).
  • If a number does not divide \(|G|\), there cannot be a subgroup of that order.

What Lagrange does not say

  • It does not say that every divisor of \(|G|\) actually occurs as a subgroup order in every group.
  • It does not prove that every element order is possible just because it divides \(|G|\).

Good proof moves with cosets

  • To prove two cosets are equal, show one representative differs from the other by an element of \(H\).
  • To count cosets in a finite group, use \(|G|/|H|\).
  • To prove a subgroup is normal, compare left and right cosets.

Common exam examples

  • In \((\mathbb{Z},+)\) with \(H=3\mathbb{Z}\), the cosets are \(0+H,1+H,2+H\). This is the additive version of congruence classes mod \(3\).
  • If \(|G|=12\) and \(|H|=4\), then \([G:H]=3\), so there are exactly three left cosets.
  • If \(|G|=21\), then no subgroup of order \(8\) can exist because \(8\nmid 21\).

Direct Products and Finite Abelian Group Structure

Direct products

If \(G_1,\dots,G_n\) are groups, then \(G_1\times\cdots\times G_n\) is a group under componentwise operation. The identity is the tuple of identities and inverses are taken componentwise.

  • \(|(g_1,\dots,g_n)|=\operatorname{lcm}(|g_1|,\dots,|g_n|)\) when the component orders are finite.
  • The product is abelian iff each factor is abelian.
  • A product of cyclic groups need not be cyclic.

When is a product cyclic?

A very common fact is that \(\mathbb{Z}_m\times\mathbb{Z}_n\) is cyclic if and only if \(\gcd(m,n)=1\). More generally, the lcm of the component orders must be large enough to equal the group order if one element is going to generate everything.

Finite abelian groups

Finite abelian groups are built from products of cyclic prime-power groups. In practical problem solving, this means that to list possible abelian groups of order \(n\), first factor \(n\) into prime powers, then write all partitions of each prime-power exponent. Each choice gives one isomorphism type.

Practical examples of classification thinking

  • Order \(8\): \(\mathbb{Z}_8\), \(\mathbb{Z}_4\times\mathbb{Z}_2\), \(\mathbb{Z}_2\times\mathbb{Z}_2\times\mathbb{Z}_2\).
  • Order \(12\): abelian possibilities are \(\mathbb{Z}_{12}\) and \(\mathbb{Z}_6\times\mathbb{Z}_2\), which is isomorphic to \(\mathbb{Z}_3\times\mathbb{Z}_2\times\mathbb{Z}_2\).
  • Order \(24\): separate the \(2^3\) part and the \(3\) part, then combine the possible \(2\)-primary pieces with \(\mathbb{Z}_3\).

How to show two groups are not isomorphic

  • Compare whether they are abelian.
  • Compare whether they are cyclic.
  • Compare numbers of elements of each order.
  • Compare subgroup structure.

Common exam examples

  • \(\mathbb{Z}_4\not\cong \mathbb{Z}_2\times\mathbb{Z}_2\) because \(\mathbb{Z}_4\) has an element of order \(4\), while \(\mathbb{Z}_2\times\mathbb{Z}_2\) does not.
  • \(\mathbb{Z}_6\cong \mathbb{Z}_2\times\mathbb{Z}_3\) because \(\gcd(2,3)=1\) and both groups are cyclic of order \(6\).
  • \(S_3\not\cong \mathbb{Z}_6\) because one is nonabelian and the other is abelian.

Group Homomorphisms, Kernels, Images, and Isomorphisms

Homomorphism definition

A map \(\varphi:G\to G'\) is a group homomorphism if \(\varphi(xy)=\varphi(x)\varphi(y)\) for all \(x,y\in G\). That is the whole definition. Injectivity and surjectivity are separate questions.

Automatic identities

  • \(\varphi(e_G)=e_{G'}\).
  • \(\varphi(x^{-1})=\varphi(x)^{-1}\).
  • \(\varphi(x^n)=\varphi(x)^n\) for every integer \(n\).
  • The image of a subgroup is a subgroup.

Kernel and image

The kernel is \(\ker\varphi=\{x\in G:\varphi(x)=e_{G'}\}\), and the image is \(\operatorname{im}\varphi=\{\varphi(x):x\in G\}\). These two objects capture most of the important information in a homomorphism problem.

  • The kernel is always a normal subgroup of the domain.
  • The image is always a subgroup of the codomain.
  • \(\varphi\) is injective iff \(\ker\varphi=\{e\}\).
  • \(\varphi\) is surjective iff \(\operatorname{im}\varphi=G'\).

Isomorphisms

An isomorphism is a bijective homomorphism. If there exists an isomorphism \(G\cong H\), then the groups are algebraically the same even if they look different on the surface. Isomorphisms preserve identity, inverses, orders of elements, subgroup structure, normality, cyclicity, and whether the group is abelian.

How to prove two groups are isomorphic

  • Give a candidate map explicitly.
  • Prove it is a homomorphism.
  • Prove it is injective and surjective, or otherwise prove bijection.

How to disprove isomorphism quickly

  • If one group is abelian and the other is not, they are not isomorphic.
  • If one has an element of a certain order and the other does not, they are not isomorphic.
  • If one is cyclic and the other is not, they are not isomorphic.

Cayley's theorem and permutation representations

Every group is isomorphic to a subgroup of a permutation group. This is a deep structural fact because it means that arbitrary groups can always be realized concretely as symmetries of a set.

First isomorphism theorem

For any homomorphism \(\varphi:G\to G'\), \[ G/\ker\varphi \cong \operatorname{im}\varphi. \] In many quotient problems, the real job is to identify the correct kernel and image and then quote this theorem cleanly.

Common homomorphism examples

  • The sign map \(\operatorname{sgn}:S_n\to\{\pm1\}\) has kernel \(A_n\), so \(S_n/A_n\cong \{\pm1\}\cong \mathbb{Z}_2\).
  • The map \(\varphi:\mathbb{Z}\to\mathbb{Z}_n\) given by \(\varphi(k)=[k]\) is surjective with kernel \(n\mathbb{Z}\), so \(\mathbb{Z}/n\mathbb{Z}\cong \mathbb{Z}_n\).
  • The determinant map \(\det:GL_n(\mathbb{R})\to\mathbb{R}^{\times}\) is a homomorphism, and its kernel is \(SL_n(\mathbb{R})\).
  • The map \(\varphi:\mathbb{R}\to\mathbb{R}_{>0}\) given by \(\varphi(x)=e^x\) is an isomorphism from \((\mathbb{R},+)\) to \((\mathbb{R}_{>0},\cdot)\).

Normal Subgroups and Quotient Groups

Normal subgroup tests

A subgroup \(N\le G\) is normal if it is stable under conjugation. The standard equivalent formulations are:

  • \(gN=Ng\) for all \(g\in G\).
  • \(gNg^{-1}=N\) for all \(g\in G\).
  • \(gng^{-1}\in N\) for all \(g\in G\) and \(n\in N\).

Quick sources of normality

  • Every subgroup of an abelian group is normal.
  • Every kernel is normal.
  • Every subgroup of index \(2\) is normal.

Quotient groups

If \(N\trianglelefteq G\), then the set of left cosets \(G/N\) becomes a group under \[ (aN)(bN)=(ab)N. \] Normality is exactly what makes this operation well defined. Without normality, different choices of representatives can produce different answers.

What to remember about quotients

  • The elements of \(G/N\) are cosets, not elements of \(G\).
  • The identity element is the coset \(N\).
  • The inverse of \(aN\) is \(a^{-1}N\).
  • If \(G\) is finite, then \(|G/N|=[G:N]=|G|/|N|\).

Quotient intuition

Taking a quotient means collapsing all elements of the same coset into one object. In effect, the subgroup \(N\) becomes the identity in the quotient. This is why quotient groups and homomorphisms are so closely related: a homomorphism identifies elements that differ by something in its kernel.

Common exam examples

  • \(\mathbb{Z}/n\mathbb{Z}\) is the standard quotient: all integers that differ by a multiple of \(n\) become the same element.
  • Since \(A_3\trianglelefteq S_3\) and \(|S_3:A_3|=2\), the quotient \(S_3/A_3\) has order \(2\), so it is isomorphic to \(\mathbb{Z}_2\).
  • If \(N\trianglelefteq G\) has index \(3\), then \(|G/N|=3\), so the quotient is cyclic of prime order.
  • In an abelian group \(G\), every subgroup \(N\) is normal, so quotients \(G/N\) always make sense.

Proof Patterns and Problem-Solving

Equivalence relation proofs

State the relation carefully. Then prove reflexive, symmetric, and transitive in that order. If one of them fails, stop and give a clean counterexample instead of wandering through extra algebra.

Group proofs

First show the operation is binary. Then check closure and associativity. Then identify the identity. Then solve for inverses. If the operation comes from something already known to be associative, say that explicitly.

Subgroup proofs

Use the subgroup test. Let \(a,b\in H\). Show \(ab^{-1}\in H\). If you can, also say why \(H\neq\varnothing\). In many proofs this is the fastest correct path.

Order arguments

Ask what the least positive exponent is that returns the identity. For permutations, use disjoint cycles. For products, use least common multiples. For finite groups, use divisibility from Lagrange.

Isomorphism arguments

To prove groups are isomorphic, define a candidate map and verify the homomorphism law. Then prove bijectivity. To prove groups are not isomorphic, compare invariants and look for a mismatch.

Homomorphism arguments

Compute \(\varphi(xy)\) and compare it directly to \(\varphi(x)\varphi(y)\). After that, determine the kernel and image. Many follow-up questions become automatic once those are known.

Normality and quotient arguments

If a quotient appears, the first sentence should usually address normality. If normality is true, then justify that the coset product is well defined. If the subgroup is not normal, then the quotient group does not exist.

When a problem asks for all possibilities

  • Use divisibility restrictions from Lagrange.
  • Use cyclic subgroup structure if the group is cyclic.
  • Use element orders and subgroup counts to rule cases out.
  • For finite abelian groups, use the classification into cyclic prime-power factors.

Common Mistakes and Corrections

  • Mistake: forgetting to show an operation is well defined on equivalence classes. Correction: prove the result does not depend on the chosen representatives.
  • Mistake: treating closure alone as enough for a subgroup. Correction: use \(ab^{-1}\in H\), or the finite subgroup test when allowed.
  • Mistake: composing permutations left to right. Correction: apply the rightmost permutation first.
  • Mistake: assuming every divisor of \(|G|\) must occur as an element order or subgroup order. Correction: Lagrange gives necessary conditions, not always existence.
  • Mistake: thinking a homomorphism must be injective or must come from abelian groups. Correction: the only defining property is preserving the operation.
  • Mistake: writing quotient groups for arbitrary subgroups. Correction: only normal subgroups give quotient groups.
  • Mistake: confusing the kernel with the image. Correction: the kernel lives in the domain and maps to the identity; the image lives in the codomain and consists of values actually hit.
  • Mistake: claiming two groups are isomorphic because they have the same size. Correction: equal size alone is far from enough.
